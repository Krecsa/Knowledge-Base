## **Сравнение с Windows и macOS**

| Назначение                  | Linux              | Windows                           | macOS                               |
| --------------------------- | ------------------ | --------------------------------- | ----------------------------------- |
| Домашняя папка пользователя | `/home/user`       | `C:\Users\user`                   | `/Users/user`                       |
| Глобальные конфиги          | `/etc`             | `C:\ProgramData`, реестр          | `/etc` → `/private/etc`             |
| Системные команды           | `/bin`, `/usr/bin` | `C:\Windows\System32`             | `/bin`, `/usr/bin` (симлинки)       |
| Блочные устройства          | `/dev/sda1`        | `\\.\PhysicalDrive0`              | `/dev/disk0s1`                      |
| Съёмные носители            | `/mnt`, `/media`   | `D:\`, `E:\`                      | `/Volumes`                          |
| Временные файлы             | `/tmp`             | `C:\Windows\Temp`                 | `/tmp` → `/private/tmp`             |
| Логи                        | `/var/log`         | `C:\Windows\Logs`, `Event Viewer` | `/var/log` → `/private/var/log`     |
| Программное обеспечение     | `/opt`             | `C:\Program Files`                | `/Applications` (GUI), `/opt` (CLI) |
| Пользовательские настройки  | `~/.config`        | `%APPDATA%`                       | `~/Library/Preferences`             |
| Кэш пользователя            | `~/.cache`         | `%LOCALAPPDATA%\Temp`             | `~/Library/Caches`                  |

## Категории пользователей и типы прав

**Категории пользователей**:

| Категория      | Кто это                            |
| -------------- | ---------------------------------- |
| **User (u)**   | Владелец файла                     |
| **Group (g)**  | Пользователи из группы владельца   |
| **Others (o)** | Все остальные пользователи системы |
**Типы прав:**

| Символ | Право   | Значение                                    | Число |
| ------ | ------- | ------------------------------------------- | ----- |
| `r`    | read    | Чтение файла или просмотр содержимого папки | 4     |
| `w`    | write   | Запись или изменение файла/папки            | 2     |
| `x`    | execute | Выполнение файла или вход в папку           | 1     |
**Примеры:**

`chmod 777 file` - Все могут читать, изменять и выполнять файл.
`chmod 755 file` - Владелец имеет все права, остальные — только чтение и выполнение.
`chmod 644 file` - Владелец может читать и изменять, остальные — только читать.

**Рекурсивное изменение:**
Рекурсивное изменение - применение команды не только к указанному файлу или директории, но и ко всем вложенным файлам и поддиректориям внутри неё.
Для рекурсивного изменения прав используется опция `-R`, например: `chmod -R 755 directory/` 

## **Сети**

**Сравнение ifconfig и ip**

| Функция                           | ifconfig                                                     | ip                                                 |
| --------------------------------- | ------------------------------------------------------------ | -------------------------------------------------- |
| **Показать все интерфейсы**       | `ifconfig -a`                                                | `ip a` или `ip link show`                          |
| **Показать конкретный интерфейс** | `ifconfig eth0`                                              | `ip a show eth0`                                   |
| **Включить интерфейс**            | `ifconfig eth0 up`                                           | `ip link set eth0 up`                              |
| **Выключить интерфейс**           | `ifconfig eth0 down`                                         | `ip link set eth0 down`                            |
| **Назначить IP-адрес**            | `ifconfig eth0 192.168.1.100 netmask 255.255.255.0`          | `ip addr add 192.168.1.100/24 dev eth0`            |
| **Удалить IP-адрес**              | `ifconfig eth0 0.0.0.0` (неявно)                             | `ip addr del 192.168.1.100/24 dev eth0`            |
| **Добавить второй IP (алиас)**    | `ifconfig eth0:1 10.0.0.1`                                   | `ip addr add 10.0.0.1/24 dev eth0 label eth0:1`    |
| **Сменить MAC-адрес**             | `ifconfig eth0 hw ether 00:11:22:33:44:55`                   | `ip link set dev eth0 address 00:11:22:33:44:55`   |
| **Посмотреть статистику**         | `ifconfig eth0`                                              | `ip -s link show eth0`                             |
| **Поддержка IPv6**                | Ограниченная (требуется `ifconfig -a` или отдельные утилиты) | Полная (`ip -6 addr show`)                         |
| **Управление маршрутами**         | Нет (нужна команда `route`)                                  | `ip route add/del/show`                            |
| **Показать ARP/соседей**          | `arp -n` (отдельная команда)                                 | `ip neigh show`                                    |
| **MTU**                           | `ifconfig eth0 mtu 1500`                                     | `ip link set eth0 mtu 1500`                        |
| **Состояние интерфейса**          | `RUNNING`, `UP`, `BROADCAST` (текстовые флаги)               | `UP/LOWER_UP` (более детально)                     |
| **Информация о NIC-драйвере**     | Нет                                                          | `ethtool -i eth0` или `ip link show` (ограниченно) |
| **Поддержка Network Namespace**   | Нет                                                          | Да (`ip netns`)                                    |
| **Цветной вывод**                 | Нет                                                          | Да (в некоторых дистрибутивах)                     |
| **Скорость работы**               | Медленнее (deprecated)                                       | Быстрее, оптимизирован                             |
| **Статус в современных Linux**    | Устаревшая (deprecated)                                      | Актуальная, рекомендуется                          |
**Ping**

| Опция          | Описание                                     |
| -------------- | -------------------------------------------- |
| `-c <число>`   | Ограничить количество запросов               |
| `-i <секунды>` | Интервал между отправкой пакетов             |
| `-s <размер>`  | Размер пакета данных в байтах                |
| `-W <секунды>` | Таймаут ожидания ответа на каждый пакет      |
| `-f`           | Беспорядочная отправка (flood), требует root |
| `-q`           | Тихий вывод (только итоговая статистика)     |
| `-4`           | Принудительно использовать IPv4              |
| `-6`           | Принудительно использовать IPv6              |
| `-D`           | Выводить время в формате UNIX timestamp      |
| `-O`           | Выводить сообщения о пропущенных пакетах     |
## Дополнительные примечания:

- **ifconfig** — часть пакета **net-tools** (не устанавливается по умолчанию во многих новых дистрибутивах, например, RHEL 8+, Ubuntu 18.04+)
    
- **ip** — часть пакета **iproute2**, устанавливается везде по умолчанию
    
- **ip** выводит информацию в более структурированном виде, что удобно для парсинга скриптами
    
- Для просмотра всех интерфейсов, включая отключённые: `ip link show`
## **Следует запомнить**

Для системных файлов нужно использовать `sudo`

**`ls`** - Выводит список файлов и папок в текущей директории

**`ls -a`** - Показывает все файлы и папки в текущей директории, включая скрытые
- **`.` (одна точка)** — это ссылка на **текущую директорию**
- **`..` (две точки)** — это ссылка на **родительскую директорию**

**`cd`** -  Перемещение между директориями

**`cat`** - Выводит последовательно указанные файлы, объединяя их в единый поток

**`cp`** - Копирование файла

`rm` - Удаление файла

**`touch`** - Создать файл в выбранной директории

**`nano`** - Текстовый редактор. **CTRL + O** для сохранения файла

**`sudo`** - Временно дает права root и записывает все действия в `/var/log/auth.log`

**`/var/log`** - Здесь можно найти логи

**`/etc/passwd`** - Здесь хранится основная информация о пользователях и логины

**Обновление:**
sudo apt update обновит пакеты  
sudo apt full-upgrade выполнит глубокое обновление  
sudo apt autoremove удалить ненужные пакеты и исправит несовместимости
sudo reboot перезагрузка
